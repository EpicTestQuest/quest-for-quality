# Beta Testing Feedback Form Questions

This document contains the feedback questions for beta testers. The questions are designed to cover Feasibility, Viability, and Desirability using a hybrid approach of quantitative and qualitative methods.

---

## Quantitative Questions

### 1. Net Promoter Score (NPS)
**How likely are you to recommend Wizzo to a colleague or friend?**

Scale: 0-10
- 0 = Not at all likely
- 10 = Extremely likely

---

### 2. Feature Value Assessment
**Please rate the value of each feature you tested (1-5 scale)**

- Test case generation (conversational flow)
- Test case generation (modal flow)
- Test case generation (image upload)
- Test case generation (GitHub PR/Issue)
- Manual test case creation
- GitHub integration (posting tests)
- Jira integration (optional)
- Quality Parties (brainstorming sessions)
- Product/Feature/Persona management
- Focus Break games

Scale:
- 1 = Not valuable
- 2 = Somewhat valuable
- 3 = Valuable
- 4 = Very valuable
- 5 = Extremely valuable

---

### 3. Usability Rating
**How easy was Wizzo to use overall?**

Scale: 1-5
- 1 = Very difficult
- 2 = Difficult
- 3 = Neutral
- 4 = Easy
- 5 = Very easy

---

### 4. Time Savings Estimate
**Compared to your current testing workflow, how much time do you think Wizzo would save you?**

Options:
- [ ] No time savings
- [ ] 1-25% time savings
- [ ] 26-50% time savings
- [ ] 51-75% time savings
- [ ] 76-100% time savings

---

### 5. Willingness to Pay (Viability)
**If Wizzo were available today, which pricing tier would you consider?**

Options:
- [ ] Free tier only
- [ ] $10-20/month per user
- [ ] $21-40/month per user
- [ ] $41-60/month per user
- [ ] $60+/month per user
- [ ] I would not pay for this tool
